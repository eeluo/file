Analyze the attribute section.

The AttributeRecognizer function determines whether the string of the attribute definitions section conforms to the specification.

Author
luoaling

AttributeRecognizer function uses regular expressions to check the attribute definitions of the string, in order to achieve the purpose of testing, regular expression matching string is relatively simple, while the use of regular expression matching the default packet order, to the corresponding Attribute class object assignment, and finally add a container that stores an Attribute type.

Parameters
[in]_lines_lines is a row in the attribute definitions section of the string.
[in]_file_descriptor_file_descriptor is formed by the corresponding file descriptor.
Returns
Returns a Boolean value that indicates whether the string conforms to the specification and conforms to the corresponding file descriptor.
Note
The Attribute class contains only the attribute definition part and does not contain the attribute value, which consists of two parts£º attribute_definition and attribute_default.

attribute_definitions = { attribute_definition } ; attribute_definition = 'BA_DEF_' object_type attribute_name attribute_value_type ';'; object_type = '' | 'BU_' | 'BO_' | 'SG_' | 'EV_' ; attribute_name = '"' C_identifier '"' ; attribute_value_type = 'INT' signed_integer signed_integer | 'HEX' signed_integer signed_integer | 'FLOAT' double double | 'STRING' | 'ENUM'[char_string{ ',' char_string }]

attribute_defaults = { attribute_default } ; attribute_default = 'BA_DEF_DEF_' attribute_name attribute_value ';'; attribute_value = unsigned_integer | signed_integer | double | char_string;

This is a regular expression for the attribute_definition section.

Since the attribute_value_type variable is more complex, the regular expression is separated here so that the regular expression can be matched for a variety of situations.

Matches the regular expression of the string except the attribute_value_type part.

Write the five regular expressions of the attribute_value_type section, respectively.

This is a regular expression for the attribute_default section.

m0 stores the matching result for attribute_definition, m1 stores the result of attribute_default.

The five cases of the attribute_value_type part are respectively regular expression matching.

Assign a value to the attribute_default part of the variable.

Partial variables for the attribute_definition assignment, also divided into five cases to assign values.

Separate the string with spaces and commas as delimiters.
